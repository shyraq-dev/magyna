# Maǵyna

Кітаптар мен оқиғаларды оқуға, жазуға және бөлісуге арналған платформа.
Next.js 16 (LTS) + Supabase, Vercel-ге деплойға дайын. Толық детальды ТТ (Профиль, Оқырман, Жазушы,
Админ/Модерация, Бейфункционалдық талаптар) негізінде салынған.

## Жазу моделі: жалғыз автор

**Тек `admin` рөліндегі бір аккаунт (сіз) жазады.** Тіркелген барлық басқа пайдаланушылар — тек оқырман:
жаза алмайды, бірақ оқиды, пікір қалдырады, рейтинг қояды, дәйексөз бөліседі, таңдаулыға қосады, офлайн
жүктеп алады, шағымданады, қолдау қызметіне жазады.

Түпнұсқа детальды ТТ-де "Оқырман" мен "Жазушы" әр пайдаланушыда қатар болатын (dual-role) деп сипатталған,
бірақ жобаның бастапқы шешімі бойынша бұл платформада тек біреу — иесі — жазады. Сондықтан төмендегі
тізімде "Жазушы модулі" тек admin аккаунтына қатысты.

## Толық функционал

### 1. Авторизация және Профиль
- Email/пароль (кемінде 8 таңба, 1 бас әріп, 1 сан) + Google OAuth
- Құпиясөзді ұмытқанда — email-ге сілтеме арқылы қалпына келтіру (`/forgot-password` → `/reset-password`)
- Профиль: аватар (дөңгелек crop, drag+zoom, макс 5 МБ), көрсетілетін есім, bio (макс 250 таңба),
  Instagram/TikTok/веб-сайт сілтемелері (`/profile/edit`)
- Никнейм бірегейлігі — DB деңгейінде `unique` шектеу + trigger арқылы автоматты профиль жасау
  (email confirmation қосулы болса да жұмыс істейді)

### 2. Оқырман модулі
- Басты бет: "Бүгінгі таңдау" (кураторлық), "Жаңа мағыналар", "Оқырмандар сүйіктісі" каруселдері
- Жанр бойынша жылдам сүзгі
- Ақылды іздеу: search-as-you-type dropdown + Postgres full-text search (`tsvector`/GIN)
- Кітап беті: мұқаба, мета-деректер, оқу уақыты бағасы, рейтинг (★1-5), "Оқуды бастау/жалғастыру"
- Оқу экраны (`Reader`): бір тап арқылы UI жасыру, қаріп өлшемі/түрі (Charter/Inter/Merriweather),
  жол аралығы, 3 тема (қараңғы/сепия/ақ), автобетбелгі
- Абзацқа пікір (әр абзацты басып пікір қалдыруға болады)
- Дәйексөз бөлісу: мәтінді белгілеп, canvas-негізіндегі дәйексөз-карточкасын жасап, Web Share API
  арқылы (мобильде) немесе PNG жүктеп алу арқылы бөлісуге болады
- Тарау соңында лайк + "Келесі тарау" батырмасы
- Ұзақ тараулар үшін lazy-loading (алдымен 40 абзац, одан кейін scroll-мен жүктеледі)

### 3. Жазушы модулі (тек admin)
- Кітап жасау: мұқаба (макс 5 МБ), атауы (макс 100), аннотация (макс 1000), тегтер, жанрлар,
  статус ("Жазылуда"/"Аяқталды")
- Rich-text редактор: қалың/көлбеу/астын сызу, туралау (`components/RichTextEditor.tsx`,
  `contentEditable` + `execCommand` негізінде — сыртқы кітапхана қажет етпейді)
- Автосақтау (әр 6 секунд), сөз санағышы
- Жариялау / черновик / **жоспарлы жариялау** (күн-уақыт белгілеп қою — Supabase `pg_cron` арқылы
  әр минут сайын тексеріліп, автоматты жарияланады)
- Статистика беті (`/write/[id]/analytics`): тарау бойынша оқылым/лайк, соңғы 7 күндік белсенділік графигі

### 4. Әкімшілік панель (`/admin`)
- Пайдаланушылар: іздеу, рөл беру, **уақытша (3/7 күн) немесе мәңгі бұғаттау** — мерзімі аяқталған
  бұғаттар `pg_cron` арқылы автоматты алынады
- Контент модерациясы: кітап өшіру, **"Бүгінгі таңдау" куратор туы** (featured toggle)
- Шағымдар кезегі: оқырмандар жіберген шағымдарды (плагиат/балағат сөз/спам/басқа) қарап, шешу/қабылдамау
- Қолдау билеттері: пайдаланушы хабарламаларына жауап беру (жауап оқырманға in-app хабарландыру түрінде барады)
- Автоматты кілтсөз сүзгісі: пікірлер мен абзац-пікірлерінде негізгі балағат сөздер жұлдызшамен ауыстырылады
  (`lib/contentFilter.ts` — тізімді өз қалауыңызша толықтырыңыз; бұл тек клиент жағындағы алғашқы сүзгі,
  түбегейлі модерация — шағым жүйесі арқылы)

### 5. Бейфункционалдық талаптар
- **Өнімділік**: ұзақ тараулар lazy-load болады (жоғарыда)
- **Офлайн режим**: кітапты IndexedDB-ге жүктеп алу (`/offline`), app-shell кэші (`public/sw.js`)
- **Қауіпсіздік**: HTTPS+JWT (Supabase Auth), парольдер bcrypt-пен хештеледі (Supabase жағында),
  парольге күрделілік талабы
- **Копи-қорғау (жұмсақ деترент)**: оқу экранында оң жақ батырма мәзірі мен `Ctrl+C` бұғатталған,
  орнына "Дәйексөз сақтау" ұсынылады. **Ескерту: бұл нақты DRM емес** — веб-браузерде мәтінді толық
  көшіруден қорғау техникалық мүмкін емес (DevTools арқылы әрқашан оқылады). Скриншот блоктау да
  вебте мүмкін емес — тек native мобильді қосымшада (`Secure Window`) шынайы жұмыс істейді
- **Хабарландырулар**: in-app, Supabase Realtime арқылы лездік (жаңа пікір, жаңа тарау, билет жауабы).
  **OS-деңгейіндегі push (FCM)** осы жобаға кірмеді — Firebase жоба құру және VAPID/service-account
  кілттерін қосу қажет; қаласаңыз, кілттеріңізді бергенде осыны қосып беремін

## Орнату

1. **Тәуелділіктерді орнату**

   ```bash
   npm install
   ```

2. **Supabase жобасын жасау**

   Жаңа жоба болса — `supabase/schema.sql` файлын толығымен SQL Editor-де іске қосыңыз.

   Бұрыннан v2 схемаңыз бар болса (алдыңғы нұсқа), ретімен іске қосыңыз:
   - `supabase/migration_002_profile_trigger.sql`
   - `supabase/migration_003_full_spec.sql`

   **pg_cron туралы ескерту**: жоспарлы жариялау мен автоматты бұғаттан босату `pg_cron`
   кеңейтуіне тәуелді. Ол Supabase-тің кейбір тарифтерінде әдепкі бойынша қосулы болмауы мүмкін
   — Dashboard → Database → Extensions бөлімінде `pg_cron`-ды қолмен қосыңыз, әйтпесе осы екі
   SQL блогы (schema.sql-дың соңында) қатемен аяқталады.

3. **Storage buckets жасау**

   Supabase Dashboard → Storage бөлімінде екі **public** bucket жасаңыз:
   - `covers` — кітап мұқабалары үшін
   - `avatars` — профиль аватарлары үшін

4. **Google OAuth баптау**

   Supabase Dashboard → **Authentication → Providers** бөлімінде **Google**-ды қосыңыз.

   Одан кейін **Authentication → URL Configuration** бөлімінде:
   - **Site URL** — `https://magyna.vercel.app` (өз доменіңіз)
   - **Redirect URLs** — `https://magyna.vercel.app/auth/callback`, `https://magyna.vercel.app/**`

5. **Орта айнымалыларын баптау**

   ```bash
   cp .env.example .env.local
   ```

   `NEXT_PUBLIC_SUPABASE_URL` мен `NEXT_PUBLIC_SUPABASE_ANON_KEY` мәндерін енгізіңіз.

6. **Локалды іске қосу**

   ```bash
   npm run dev
   ```

7. **Vercel-ге деплой**

   Vercel Project Settings → Environment Variables бөлімінде 5-қадамдағы екі айнымалыны қосыңыз
   (Production **және** Preview орталарына).

## Өзіңізді admin етіп тағайындау

```sql
update public.profiles set role = 'admin' where username = 'сіздің_никнеймиңіз';
```

```
app/
  (auth)/login, register, forgot-password, reset-password
  books/[id]                 — кітап беті + Reader
  offline/                    — офлайн кітапхана мен оқу режимі
  write/[id]/analytics         — жазушы статистикасы
  write/                        — жазушы кабинеті (тек admin)
  admin/                         — әкімшілік панель (шағым, билет, пайдаланушы, контент)
  profile/[id], profile/edit      — профиль көрінісі мен баптау
  support/                         — қолдау билеттері (пайдаланушы жағы)
components/
  Reader.tsx                  — оқу экраны (фонттар, абзац-пікір, дәйексөз, lazy-load)
  RichTextEditor.tsx           — жазушы редакторы
  AvatarCropper.tsx              — аватар crop
  ReportButton.tsx, StarRating.tsx, Carousel.tsx, GenreFilter.tsx, LiveSearch.tsx
  admin/                            — AdminUsers, AdminBooks, AdminGenres, AdminReports, AdminTickets
lib/
  offline.ts                  — IndexedDB (офлайн оқу)
  paragraphs.ts, quoteImage.ts, contentFilter.ts, text.ts, validation.ts
supabase/
  schema.sql                  — толық v3 схема (жаңа жобаға)
  migration_002_profile_trigger.sql, migration_003_full_spec.sql — қадамдық жаңартулар
```
